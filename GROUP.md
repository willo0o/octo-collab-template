# GROUP.md - 群规与协作规范

> 协作共识 · 行为边界 · 角色职责

---

## 🎯 协作原则

1. **任务隔离**：每任务独立目录，减少冲突
2. **文档即代码**：所有协作内容用 Git 版本控制
3. **小步快跑**：频繁提交，小步迭代
4. **清晰可追溯**：每次修改有记录、可回溯
5. **人虾平等**：龙虾与人类成员享有同等的提交权限

---

## 👥 成员行为规范

### 人类成员
- 创建任务时必须填写完整的 brief.md（目标 + 截止日 + 验收标准）
- 每日更新任务状态（进行中的任务）
- 重要决策必须记录到 decisions.md
- 交付物必须归档到 deliverables/ 目录
- 提交前先 `git pull`，避免冲突

### 龙虾成员
- 未被 @ 时不主动发言（NO_REPLY）
- 被 @ 时第一时间响应
- 会议中自动记录纪要，但不打断发言
- 发现风险主动 @负责人预警
- 所有输出必须归档到 Git，不能只在群里发送

---

## 🚨 禁止行为

- ❌ 未经确认删除他人任务目录
- ❌ 直接修改他人 brief.md（应走 PR 或先沟通）
- ❌ 在群内发送无信息量消息
- ❌ 交付物只在群聊发送，不归档到 Git
- ❌ 任务状态长期不更新（>3 天）

---

## 🔧 工具约定

| 工具 | 用途 |
|------|------|
| Git / GitHub | 任务跟踪、文档协作 |
| 群聊 | 实时沟通、@提醒 |
| `Scripts/create-task.sh` | 快速创建新任务 |
| `TODO.md` | 任务主索引，快速查看全局进展 |
